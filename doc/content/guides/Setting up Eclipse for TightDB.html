Setting up Eclipse for TightDB

Open the project in which you want to use TightDB and open the Properties dialog by selecting the 'Project → Properties' menu item.

  1. Make sure the Java compiler is configured with 1.6 or 1.7 version.
    • Go to the 'Java Compiler' page.
    • The configured compiler version should be at least 1.6.

  2. Add tightdb.jar to the build path.
    • Go to the 'Java Build Path' page.
    • Select the 'Libraries' tab.
    • Click the 'Add External JARs...' botton.
    • Select tightdb.jar. It will usually be found in /usr/local/share/java, but your local setup may vary. Check the output of the install process for precise location.

  3. Configure tightdb-devkit.jar as annotation processor.
    • Go to the 'Java Compiler → Annotation Processing' page.
      • Check 'Enable project specific settings.
      • Enable annotation processing (check both check-boxes).
      • Configure the output folder for the generated sources in the "Generated source directory" field.
      • If the project sources are in folder(s) different than "src", "src/main/java" and "src/test/java", the path to the source files must be specified.
    • Go to 'Java Compiler → Annotation Processing → Factory Path' page.
      • Check 'Enable project specific settings.
      • Click the 'Add External JARs...' botton.
      • Add tightdb-devkit.jar (found in same location as tightdb.jar) and make sure it is checked.

If the previous steps was done properly, Eclipse will start generating Java source code as soon as you annotate your schema specification classes with the @DefineTable annotation. Whenever you save a file, the TightDB files are regenerated.
