TightDB example instructions

Below are instructions for testing the TightDB example applications in Java using Ant or Eclipse.

Integrating Tightdb with build tools and IDE's

If you wish to integrate TightDB in a new or existing application using Maven or Ant (and any IDE), please see reference setups and instructions for Maven Setup or Ant Setup.

If you wish to configure Eclipse for optimal TightDB experience, please read Eclipse Setup.

Installing TightDB

After installing TightDB, ensure you have Java JDK6 or JDK7 installed: 

$ javac -version
javac 1.7.0_07

If the command is not available, please install your preferred JDK.

Building TightDB for Java produces 3 main components:

  • libtightdb-jni.{so, libjni} - a native, binary TightDB library,
  • tightdb.jar - the TightDB Java run-time that uses the native library,
  • tightdb-devkit.jar - TightDB's APT-based tool for annotation-driven code generation at compile-time (NOT needed at run-time).

Below instructions for running the examples works whether you have only built TightDB or if you have also installed it.

TightDB example applications

The TightDB installation contains a folder with Java examples: 
cd tightdb_java/examples/intro-example
It contains 4 examples:
  • "tutorial", which is the exact code you see in the tutorial documentation.
  • "showcase", which shows a a little more of TightDB's features.
  • "quickbenchmark", which tests TightDB against native Java ArrayList
  • "performance", which tests TightDB, native Java ArrayList and SQLite in a simple performance test.

Running TightDB examples using Ant

If you have Ant installed, you can build and run the examples from your shell as follows:

Build examples:

ant build

Run examples:

ant tutorial
ant showcase
ant quickbenchmark
ant performance

Generating TightDB classes

ant generate-sources

Running TightDB examples using Eclipse

If you wish to try the examples in the Eclipse IDE instead,

Build

  • Open eclipse and select an existing or new workspace directory.
  • Import example
    • Select "File → Import".
    • Choose "General → Existing Projects into Workspace".
    • Select root directory: Browse for the "intro-example" folder and select it.

Run

  • Select "Run → Run", "Run as Java Application" and select "tutorial" (or another example)

Generating TightDB classes

  • You can now change the examples as you wish. If you change or add TightDB tables with the @DefineTable annotation, classes are automatically generated for you when you save a file. The generated classes will be available with full auto-completion etc.
