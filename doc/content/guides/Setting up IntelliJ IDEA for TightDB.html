Setting up IntelliJ IDEA for TightDB

  1. Make sure the Java compiler is configured with 1.6 or 1.7 version.
    • Go to File → Project Structure → SDK.
    • Make sure the configured compiler version is at least 1.6.
  2. Make sure tightdb.jar and tightdb-devkit.jar are added to the build path.
  3. Configure the annotation processor (this will enable IntelliJ IDEA to generate the TightDB-related source code).
    • Go to Preferences → Compiler → Annotation Processors
    • Check "Enable annotation processing"
    • Check "Obtain processors from project classpath"
    • Make sure that "Store generated sources relative to" is set to "Module content root"
    • In "Production sources library" and "Test sources library" "configure the output folders, e.g. "generated" and "generated-test"."
  4. Add "generated" and "generated-test" as Sources and Test Sources"
    • Go to File → Project Structure → Modules
    • Mark "generated" as Sources
    • Mark "generated-test" as Test Sources
  5. When the above steps is completed IntelliJ IDEA will generate sources from classes annotated with @DefineTable every time you compile your project. After compilation it might be needed to Synchronize your project, before the ide acknowledges the new sources.
